# このプロジェクトについて

masawadaの3DモデルをAIが自動で踊らせるスクリーンセーバー。
Three.js + three-vrm で動く単一HTMLファイル構成。

## 実装方針

- すべて `index.html` 1ファイルに収める
- ライブラリはCDN（Three.js 0.108）と同梱（three-vrm.min.js）
- Three.js 0.108 の古いAPI: `addAttribute`（`setAttribute`ではない）

## デザインポリシー

### やってはいけないこと

- **半透明の図形を重ねる** → だらしなく見える。図形を使うなら単色・不透明で
- **ふわふわ浮遊するパーティクル** → 遅くて間延びして見える
- **縦の雨エフェクト** → ダサい
- **ファミコン風の太い走査線** → レトロゲームエミュレータみたいになる。走査線を入れるなら1px・低不透明度・サイン波で輝度を変化させて「縞」でなく「質感」にする
- **ワイヤーフレームを長時間表示** → 細かすぎて不気味になる
- **UIテキストを画面に常時表示** → スクリーンセーバーとしてノイズになる

### やるべきこと

- **2Dオーバーレイはキメ（質感）重視** → フィルムグレイン・グリッチバーのような硬いグラフィック
- **エフェクトはベタ塗り単色で瞬間的に** → ビートフラッシュは一瞬だけ全面を塗って即切る
- **動きは唐突に** → EMAスムージングで普段は滑らかにしつつ、スナップポーズで急に違う形に飛ぶ
- **走査線はあってもなくてもわからない程度** → 見えるか見えないかぐらいが正解

## アーキテクチャメモ

- `AudioSystem` … ビートシミュレーター（マイク入力なし）
- `CameraSystem` … 18種のMVショット、加重ランダム切り替え
- `DanceEngine` … 5スタイル、EMAスムージング、スナップポーズシステム
- `WorldGenerator` … 11種のプロシージャルステージ
- `OverlaySystem` … canvas2d による2Dエフェクト（グレイン・グリッチ・ビートフラッシュ）
- ダンサーはプール3体、同時に最大3人まで有効化
- ビートで `triggerSnap()` を呼ぶと8種のポーズに瞬間スナップ
